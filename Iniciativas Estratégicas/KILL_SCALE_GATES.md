# Compuertas Kill / Scale por PLAN

> **STATUS:** current
> **CANONICAL_ARCHITECTURE:** 22 thematic + PLANRUTA protocol
> **REGISTRY:** ver `PLAN_REGISTRY.yml`
> **PRINCIPIO:** cada PLAN tiene 3 gates con 5 criterios binarios cada uno + 1 condición de kill automática.
> **VINCULANTE:** sí. El PEO no firma promoción de un PLAN sin gate cerrado y validado.
> **LAST_AUDIT:** 2026-04-26

## Convención

- **Gate 1 — Entrada a piloto:** condiciones para que el PLAN salga de diseño y comience operación piloto.
- **Gate 2 — Piloto a escala:** condiciones para expandir cobertura geográfica/sectorial.
- **Gate 3 — Escala a sistema:** condiciones para integrar como capacidad permanente del Estado.

Cada gate requiere los 5 criterios marcados + ausencia de condición de kill.

## Reglas de promoción entre PLANes

1. Un PLAN solo puede avanzar al siguiente Gate si el actual cumplió **5 criterios + ausencia de KILL durante 24 meses**.
2. El PEO firma la promoción.
3. La promoción se publica en el dashboard público.
4. Si un PLAN activa KILL, el PEO convoca revisión extraordinaria.
5. Si un PLAN no avanza Gate en 18 meses sin justificación, se evalúa freeze.
